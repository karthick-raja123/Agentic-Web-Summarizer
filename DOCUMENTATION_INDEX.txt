## Key Metrics

### Latency (Speed)
- Measured in milliseconds (ms)
- Lower is better
- Improvement shown as percentage or multiplier (e.g., 3.4x faster)

### Cost (Budget)
- Measured in USD
- Calculated from token usage + pricing
- Savings shown as percentage (e.g., 73% cheaper)

### Quality (Correctness)
- Scored 0-100%
- Target: >= 70%
- Components: completeness, accuracy, relevance

### Cache Hit Rate
- Percentage of queries served from cache
- Higher is better
- Typical: 30-60% with learning
